Abstract

Current AI agents are constrained by manually predefined tool libraries and rigid function-calling protocols. We propose a paradigm-shifting vision: transforming GitHub's 28 million code repositories into the world's largest, most dynamic adaptive tool ecosystem. RepoMaster achieves this through revolutionary cognitive architecture that mimics human programmer patterns - building hierarchical understanding, strategic exploration, and intelligent component prioritization. Our system addresses four core technical bottlenecks: structural understanding deficiencies, context management failure, inefficient code reuse, and insufficient dependency tracking. Through hybrid hierarchical structure modeling, progressive context-aware exploration, and core component priority strategies, RepoMaster enables agents to automatically understand and interact with any repository without manual engineering. RepoMaster achieves a 95.45% valid submission rate and 27.27% medal acquisition rate on MLE-Lite benchmark, while on GitTaskBench, it improves task pass rate from 24.07% to 62.96% and reduces token consumption by 95%. This breakthrough establishes the first truly scalable, self-evolving AI tool ecosystem that aggregates the collective wisdom of millions of global developers.

Experimental Results

We evaluate RepoMaster on two challenging benchmarks: MLE-Lite and GitTaskBench. Our experiments demonstrate significant improvements over state-of-the-art baselines including OpenHands, SWE-Agent, across multiple LLMs (GPT-4o, Claude 3.5, DeepSeek V3). On MLE-Lite, RepoMaster with Claude 3.5 achieves the highest performance with 95.45% valid submission rate and 27.27% medal acquisition rate. On GitTaskBench, RepoMaster significantly outperforms baselines in both task completion and token efficiency.

üéØ Technical Challenges Solved

Existing frameworks face four core technical bottlenecks that RepoMaster systematically addresses:

Structural Understanding Deficiencies

Frameworks like OpenHands and SWE-Agent fail to effectively parse complex repository architectures, often drowning in irrelevant code while missing critical components.

Context Management Failure

Faced with intricate dependency relationships and information density exceeding LLM context windows, existing agents lack strategic information management capabilities.

Inefficient Code Reuse

Mainstream approaches rely on generating code from scratch, ignoring GitHub's rich ready-made solutions.

Insufficient Dependency Tracking

Lack of mechanisms for deep repository structure understanding and fine-grained dependency tracking.
